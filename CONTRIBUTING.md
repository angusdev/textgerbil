# Contributing to TextGerbil

Welcome! We appreciate your interest in contributing to TextGerbil. While the project is maintained by a human maintainer, code changes are generated using AI agents. As a result, we follow a unique contribution workflow.

## The Contribution Workflow

To maintain the architectural integrity of our single-file codebase and avoid complex merge conflicts with parallel AI edits, we generally **do not merge large or structural Pull Request code directly**.

Small, low-risk changes (e.g., minor bug fixes or trivial adjustments) may be merged directly if they are easy to review and clearly understood.

For larger or more complex changes, PRs are used as a blueprint. In these cases, the maintainer may not be able to fully understand or confidently review substantial code modifications. If your proposal is accepted, the maintainer will use your provided prompt and instructions to generate the change via an AI agent and apply it locally, instead of merging the submitted diff.

If a PR is not merged directly, it will be **Closed** and labeled `merged-via-prompt` to indicate that the change has been accepted and merged through prompt-based regeneration.

### How to Submit a Change

1.  **Draft Your Change**: Implement the feature or bug fix as you normally would. Ensure it follows the guidelines in [SPEC.md](SPEC.md).
2.  **Verify with Tests**: Run the test suite (`npm test`) and ensure all cases pass. Add new test cases for your features if applicable.
3.  **Prepare a Single-Shot Prompt**: Create a comprehensive, standalone prompt that an AI agent can use to implement your changes from scratch on the current codebase (required for larger changes).
4.  **Open a Pull Request**: Use the provided PR template to document:
    *   What the change does.
    *   The **exact prompt** to reproduce the change.
    *   The **Agent/Model** that produced the reference code.
    *   The **Test Cases** used for verification.

### Code Style & Standards

*   **Single File Control**: Keep all logic inside `index.html` as per the spec.
*   **Aesthetics**: We prioritize a premium, modern look. Avoid basic styling.
*   **Linting**: Always run `npm run lint:fix` before submitting.

---

**Note:** For larger changes, the diff in your PR is for reference only and serves as a proof of concept. The final implementation will be generated by the maintainer using your provided instructions.
